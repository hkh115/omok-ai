# Omok AI

Python으로 구현한 오목 AI 프로그램입니다. 

## Overview

이 프로젝트는 오목 게임이서 사용할 수 있는 AI를 구현하고, 암호화 및 복호화 기능을 추가해 암호학 기초를 탐구하는 것을 목표로 한 프로그램입니다.
AI는 보드의 상태를 분석하여 최적의 수를 선택하며, 탐색 알고리즘을 통해 가능한 수를 평가합니다.

## Omok

### Game Logic
오목 게임의 기본 규칙을 기반으로 보드를 구성하고 플레이어와 AI가 번갈아가며 수를 두도록 구현했습니다.
pygame을 통해 19x19 오목판을 구현하고, 플레이어가 클릭한 위치에 수를 두고, AI가 이후 수를 두도록 설계했습니다.

### Minimax Algorithm
AI는 minimax 알고리즘을 사용하여 수를 탐색합니다. 각 수에 대한 보드 상태를 평가하고, 가장 유리한 선택을 결정합니다.
- Minimax 알고리즘은 재귀함수 형태로 이루어져 있고, 최대 탐색 깊이는 5로 제한하였습니다.
- 탐색 효율을 최적화 하기 위해 메모이제이션 기법을 사용하였으며, 이는 LRU캐시를 통해 구현했습니다.
- 이미 놓여진 수를 바탕으로 맨해튼 거리가 2 이하인 수들 중, 평가 함수로 평가한 점수가 높은 상위 7개 수만 후보 수로 고려했습니다.

### 강제수
AI가 Minimax 알고리즘만으로 탐색을 한다면 시간이 너무 걸리며, 무조건 가장 좋은 수를 둘 거라는 보장이 없습니다.
- 비어있는 수를 중심으로 각 방향을 탐색해 연속된 수의 개수를 세고, 이를 통해 강제 수인지 여부를 판단합니다.
- 이 함수는 강제수(반드시 이기는 수) 뿐만 아니라 다음 차례에 이기는 수, 혹은 위협 수도 판별합니다.
- 게임을 어느 정도 시행하면 대부분 강제수 함수가 실행되고, 강제수나 위협수가 없는 경우에만 minimax알고리즘을 사용합니다.

### Alpha-Beta-Pruning
탐색 과정에서 불필요한 가지를 제거하여 탐색 효율을 높였습니다.
- alpha 값: 지금까지 Max가 확보한 "최대 점수" (Max는 이보다 나쁜 선택은 하지 않는다)
- beta 값: 지금짜지 Min이 확보한 "최소 점수" (Min은 이보다 좋은 수는 허용하지 않는다)
- 상대가 절대 선택하지 않을 수라면 더 탐색할 필요가 없다 -> 가지치기를 통해 하위 트리는 제거

### Board Evaluation
AI는 보드의 상태를 평가하여 각 수의 가치를 계산합니다.
- 각 경우마다 부여하는 점수가 다르며, 인간의 점수를 더 높게 계산해 AI가 수비를 우선시하도록 했습니다.
- 열린 사, 열린 삼 등의 패턴에 따라 다르게 점수를 부여했습니다.
- 이 함수는 후보 수 정렬과 minimax알고리즘에 모두 사용되어 AI의 의사결정에 큰 부분을 차지합니다.

## Encryption/Decryption
이 프로그램에는 기보 데이터를 보호하기 위한 암호화 및 복호화 기능이 포함되어 있습니다.
- 이 프로그램에서는 기보 데이터를 AES로 암호화하고, 그 키(AES 키)를 다시 RSA로 암호화하는 하이브리드 암호화 방식을 사용했습니다.
- AES-GCM모드가 제공하는 무결성 검증과, PSS 서명과 OAEP 패딩을 사용해 더욱 보안성을 높였습니다.
- SHA-256해시를 사용했습니다. (해시는 데이터 변조 여부 확인에 유용, 데이터 전체 대신 해시 값 비교하는게 더 빠름)
- 암호화 순서:
  1. 기보 입력
  2. 기보 데이터를 JSON으로 직렬화한 후 UTF-8로 인코딩해 바이트로 변환
  3. 바이트 데이터를 SHA-256를 사용해 해시 객체를 만들고, PSS 서명 객체를 통해 서명을 함 (PSS는 무작위 salt 값을 사용해 서명이 매번 다름)
  4. 256비트의 AES 키를 생성한 후, AES-GCM모드(내부적으로는 CTR모드와 인증 태그 사용)를 사용해 tag와 nonce를 생성(tag는 무결성 검증, nonce는 블록 암호화 시작점 역할)
  5. 대칭키를 RSA-OAEP로 암호화(OAEP는 RSA 암호화 과정에서 패딩을 생성함으로서 무작위성을 추가해 암호문 예측 불가능성을 보장함)
  6. 암호화된 패키지에 암호문, nonce, tag, AES키(암호화 됨)를 base64로 인코딩해 JSON으로 변환
  7. 서명과 암호화 패키지를 하나로 묶어 저장
- 복호호 순서:
  1. 기보 파일을 JSON으로 불러옴
  2. 각 항목을 순회해 entry에서 암호화 패키지와 필요한 데이터 불러옴
  3. 암호화 문자열을 base64로 디코딩, RSA 개인키로 enc_key를 복호화해 AES 키를 획득
     -> GCM객체를 생성하고, AES-GCM 내부에서 무결성 검증
  4. 목호화된 바이트와 서명을 입력받고, SHA256 해시를 생성해 PSS로 서명 검증
  5. 결과 정리 후 출력
