# Omok AI

Python으로 구현한 오목 AI 프로그램입니다. 

## 개요

이 프로젝트는 오목 게임서 사용할 수 있는 AI를 구현하고, 암호화 및 복호화 기능을 추가해 암호학 기초를 탐구하는 것을 목표로 한 프로그램입니다.
AI는 보드의 상태를 분석하여 최적의 수를 선택하며, 탐색 알고리즘을 통해 가능한 수를 평가합니다.

## 오목

### 게임 로직
오목 게임의 기본 규칙을 기반으로 보드를 구성하고 플레이어와 AI가 번갈아가며 수를 두도록 구현했습니다.
pygame을 통해 19x19 오목판을 구현하고, 플레이어가 클릭한 위치에 수를 두고, AI가 이후 수를 두도록 설계했습니다.

### Minimax 알고리즘
AI는 minimax 알고리즘을 사용하여 수를 탐색합니다. 각 수에 대한 보드 상태를 평가하고, 가장 유리한 선택을 결정합니다.
- Minimax 알고리즘은 재귀함수 형태로 이루어져 있고, 최대 탐색 깊이는 5로 제한하였습니다.
- 탐색 효율을 최적화 하기 위해 메모이제이션 기법을 사용하였으며, 이는 LRU캐시를 통해 구현했습니다.
- 이미 놓 수를 바탕으로 맨해튼 거리가 2 이하인 수들 중, 평가 함수로 평가한 점수가 높은 상위 7개 수만 후보 수로 고려했습니다.

### 강제수
AI가 Minimax 알고리즘만으로 탐색을 한다면 시간이 너무 걸리며, 무조건 가장 좋은 수를 둘 거라는 보장이 없습니다.
- 비어있는 수를 중심으로 각 방향을 탐색해 연속된 수의 개수를 세고, 이를 통해 강제 수인지 여부를 판단합니다.
- 이 함수는 강제수(반드시 이기는 수) 뿐만 아니라 다음 차례에 이기는 수, 혹은 위협 수도 판별합니다.
- 게임을 어느 정도 시행하면 대부분 강제수 함수가 실행되고, 강제수나 위협수가 없는 경우에만 minimax알고리즘을 사용합니다.

### 알파-베타 가지치기
탐색 과정에서 불필요한 가지를 제거하여 탐색 효율을 높였습니다.
- alpha 값: 지금까지 Max가 확보한 "최대 점수" (Max는 이보다 나쁜 선택은 하지 않는다)
- beta 값: 지금까지 Min이 확보한 "최소 점수" (Min은 이보다 좋은 수는 허용하지 않는다)
- 상대가 절대 선택하지 않을 수라면 더 탐색할 필요가 없다 -> 가지치기를 통해 하위 트리는 제거

### 보드 평가
AI는 보드의 상태를 평가하여 각 수의 가치를 계산합니다.
- 각 경우마다 부여하는 점수가 다르며, 인간의 점수를 더 높게 계산해 AI가 수비를 우선시하도록 했습니다.
- 열린 사, 열린 삼 등의 패턴에 따라 다르게 점수를 부여했습니다.
- 이 함수는 후보 수 정렬과 minimax알고리즘에 모두 사용되어 AI의 의사결정에 큰 부분을 차지합니다.

## 암호화/복호화
이 프로그램에는 기보 데이터를 보호하기 위한 암호화 및 복호화 기능이 포함되어 있습니다.
하이브리드 암호화 방식을 사용하여 데이터의 기밀성과 무결성을 동시에 보장합니다.

### 사용한 암호화 기술
- **AES-GCM**: 기보 데이터를 암호화하는 역할, 갈루와 카운터 모드(GCM)을 사용
- **PKCS1_OAEP**: OAEP(Optimal Asymmetric Encryption Padding), 선택 암호문 공격 방지하는 패딩 방식
- **PSS**: RSA암호화에서 사용되는 서명 방식, 데이터 무결성과 발신자 검증에 사용
- **SHA-256**: 데이터 해시 생성, 서명 검증에 사

### 암호화 과정
1. 기보 데이터를 입력
2. 기보 데이터를 JSON 형식으로 직렬화한 후 UTF-8로 인코딩하여 바이트 데이터로 변환
3. 변환된 데이터를 SHA-256 해시로 생성하고, RSA-PSS 방식으로 서명을 생성
4. 256비트 AES 키를 생성
5. AES-GCM 모드를 사용하여 데이터를 암호화하며, 이 과정에서 nonce와 인증 태그(tag)가 생성  
   - nonce: 암호화 시작 지점을 위한 값  
   - tag: 데이터 무결성 검증을 위한 인증 태그
6. 생성된 AES 키는 RSA-OAEP 방식으로 암호화
7. 암호문, nonce, tag, 암호화된 AES 키를 Base64로 인코딩한 후 JSON 형태로 저장
8. 암호화 패키지와 서명을 함께 저장

### 복호화 과정
1. 기보 파일을 JSON 형식으로 불러옴
2. 각 항목에서 암호화된 패키지와 필요한 데이터를 추출
3. Base64로 인코딩된 데이터를 디코딩
4. RSA 개인키를 사용하여 암호화된 AES 키를 복호화
5. 복호화된 AES 키와 nonce, tag를 사용하여 AES-GCM으로 데이터를 복호화하고 ,이 과정에서 GCM 모드가 자동으로 무결성 검증을 수행
6. 복호화된 데이터와 서명을 이용해 SHA-256 해시를 생성
7. RSA-PSS 방식으로 서명을 검증하여 데이터가 변조 여부 확인
8. 검증이 완료된 데이터를 정리하여 출력
